# AP14 – Methodenwahl

Status: abgeschlossen

## Ziel

Die Gruppe hat eine geeignete Entwicklungsmethodik für das Projekt identifiziert, bewertet und ausgewählt.

## Gewählte Methode

Für das Projekt wurde eine leichtgewichtige, Kanban-orientierte Entwicklungsmethodik mit iterativer Anpassung gewählt.

Die Aufgaben werden in ClickUp gesammelt, priorisiert und über Statusspalten wie **To Do**, **In Bearbeitung**, **Review** und **Erledigt** verfolgt. Dadurch ist sichtbar, welche Aufgaben offen sind, woran gerade gearbeitet wird und welche Ergebnisse bereits abgeschlossen wurden.

Die technische Umsetzung wird über GitHub versioniert. Änderungen können dadurch nachvollziehbar über Branches, Commits und Pull Requests bearbeitet werden. GitHub unterstützt außerdem die Nachvollziehbarkeit des Projektfortschritts und erleichtert die gemeinsame Arbeit an Code, SQL-Dateien und Dokumentation.

## Kommunikations- und Abstimmungswege

| Werkzeug / Rahmen | Zweck im Projekt |
|---|---|
| ClickUp | Aufgabenverwaltung und Kanban-Board |
| GitHub | Versionierung, Branches, Pull Requests und Nachvollziehbarkeit |
| Discord | Fachliche Teamkommunikation, Dateien, Materialien und Diskussionen |
| WhatsApp | Schnelle organisatorische Kurzabsprachen |
| Canva | Gemeinsame und parallele Bearbeitung der Präsentationen |
| Praktikum | Regelmäßiges Feedback, Kontrolle und Abstimmung mit Prof. / Betreuung |

Discord dient als zentrale fachliche Kommunikationsplattform für Abstimmungen, Materialien und Diskussionen. WhatsApp wird nur für schnelle organisatorische Kurzabsprachen genutzt.

Zusätzlich wird das regelmäßige Praktikum als Feedback- und Kontrollpunkt verwendet. Dort werden Entscheidungen, Fortschritt und offene Probleme mit der Betreuung abgeglichen.

## Präsentationen mit Canva

Präsentationen werden gemeinsam in Canva erstellt, damit mehrere Teammitglieder gleichzeitig und parallel an den Folien arbeiten können.

Dieses Vorgehen wurde gemeinsam in einem Praktikum besprochen und als Arbeitsweise für die weiteren Präsentationen festgelegt.

Leonardo Parrino / Ph4ntomic legt die jeweilige Präsentation mit Canva Pro an und lädt alle beteiligten Teammitglieder zur Bearbeitung ein. Anschließend wird im Team aufgeteilt, wer welche Inhalte vorstellt.

Die zuständigen Personen:

1. arbeiten ihren Vortragsteil anhand der in GitHub gepflegten Projektdokumentation aus,
2. übertragen die wesentlichen Inhalte in die zugehörigen Canva-Folien,
3. stimmen Folientext und Vortrag mit dem restlichen Team ab,
4. präsentieren ihren vorbereiteten Abschnitt im Vortrag.

Damit bleibt GitHub die fachliche Dokumentationsquelle, während Canva für die gemeinsame visuelle Ausarbeitung und Vorbereitung der Präsentation verwendet wird.

## Begründung der Methodenwahl

Die Methode wurde gewählt, weil das Projekt mehrere parallele Arbeitsbereiche umfasst:

- Datenbank
- Stored Procedures
- BPMN
- Webanwendung
- Dokumentation
- Präsentation

Ein klassisches Wasserfallmodell wäre für diesen Projektumfang zu unflexibel, da sich Anforderungen und Schwerpunkte durch Feedback während des Semesters verändern können.

Ein vollständiges Scrum-Vorgehen wäre für die Gruppengröße und den Projektumfang zu aufwendig. Kanban mit iterativen Feedbackschleifen passt besser zum Semesterprojekt, weil Aufgaben flexibel priorisiert, parallel bearbeitet und nach Rückmeldungen angepasst werden können.

## Ergebnis

Für das Projekt wurde eine passende Projektmethodik festgelegt. Die Arbeit wird über ein Kanban-orientiertes Vorgehen organisiert, technisch über GitHub nachvollziehbar versioniert und regelmäßig durch fachliche Abstimmung im Team sowie Feedback im Praktikum überprüft.
